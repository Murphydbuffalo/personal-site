Work

API development at Raizlabs

While at Raizlabs I lead development of a JSON API using Node.js, Express, Redis, and MySQL to support Android, iOS, and web clients for a leading fitness company's new in-home workout platform.

Development highlights of this project include:

  • Performance auditing and optimization of both the application and database layers.
  • Modern (ES6 via Babel) JavaScript written in a modular, functional style.
  • Adding services to, and interfacing with, the client's existing microservices architecture.
  • Excellent integration test coverage written in Mocha, and building on CircleCI.
  • API uptime monitoring with Runscope.
  • Stateless authentication using JSON Web Tokens.

Enterprise web app development and ETL at DrugDev

At DrugDev I was one of the principal developers of an enterprise solution for distributing urgent safety-related notifications to participants in clinical trials, including a process to migrate data from the existing solution of one of the world's 5 largest pharmaceutical companies to the new DrugDev application.

Development highlights of this project include:

  • Enforcing a large and complex set of business rules dictating who should receive these notifications, when they should receive them, and what the recipients should be permitted to do with the letter (read-only, acknowledge receipt, distribute to other users, etc).
  • Crafting a solution that lived inside DrugDev's existing 8+ year old Ruby on Rails codebase, which constrained our choice in what technologies and patterns we could apply.
  • Use of multiple DelayedJob queues to process the incoming legacy data, and import it into the DrugDev database.

Success in a start-up environment at Kitewheel

At Kitewheel I was a member of a 3, and later 4, person development team charged with building out the company's product. We were given a large degree of freedom in how we wanted to structure the application, what technologies we wanted to use, and what practices we as an engineering team would adopt.

As a junior member of this team I was able to successfully advocate for the use of several impactful best practices including:

  • A robust automated test suite (none existed before my arrival).
  • Replacing MongoDB with PostgreSQL as our persistence layer. Our data was not close to approaching the scale required by a noSQL solution, and we benefitted greatly from the consistency guarantees, ability to make ad hoc queries, and transaction support afforded by a SQL solution.
  • Use of a security best practices module, Helmet.js, to close gaps in our Node.js server's security features. Unlike a more full-featured web framework such as Rails, our Node and Express.js server was missing key protections such as CSRF tokens, a Content Security Policy, and HSTS.
